# Очистка кэша Vercel

## Проблема

Vercel может использовать кэшированную версию файлов, даже если код исправлен в репозитории.

## Решения

### Вариант 1: Очистка кэша в Vercel Dashboard

1. Зайдите в Vercel Dashboard
2. Выберите проект `vo-deco-alpha`
3. Перейдите в **Settings** → **General**
4. Прокрутите вниз до секции **"Build & Development Settings"**
5. Нажмите **"Clear Build Cache"** или **"Redeploy"**

### Вариант 2: Принудительный передеплой

1. В Vercel Dashboard → **Deployments**
2. Найдите последний деплой
3. Нажмите **"..."** → **"Redeploy"**
4. Выберите **"Use existing Build Cache"** = **OFF**
5. Нажмите **"Redeploy"**

### Вариант 3: Новый коммит (уже сделано)

Я уже отправил новый коммит `fc0dcda`, который должен запустить новый деплой.

## Проверка

После передеплоя проверьте:
1. Логи в Vercel Dashboard → Functions → Logs
2. Убедитесь, что нет ошибок про `_sum`
3. Сборка должна пройти успешно

## Если ошибка все еще есть

Возможно, проблема в другом месте. Проверьте:
1. Все ли коммиты отправлены в GitHub
2. Правильный ли коммит использует Vercel
3. Нет ли других файлов с `_sum` и `amount`
