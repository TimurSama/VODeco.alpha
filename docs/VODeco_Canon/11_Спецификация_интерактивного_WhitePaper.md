# Спецификация интерактивной страницы WhitePaper (Smart WhitePaper)

Цель: интерактивная страница должна быть “живой картой смысла” — **слои → механики → доказательства → калькуляторы → ссылки на код/источники**.

## 1) Формат: scroll‑story + якорные разделы
Рекомендуемый UX из материалов репозитория:
- “космос → Земля → подсветка воды” (вступление)
- слои (physical → data → trust → twins → economy → governance → culture)
- интерактивы: регионы/факты, переключение слоёв, раскрывающиеся карточки

Источники формулировок и сценариев: `архитектура и дополнения.txt`.

## 2) Структура (предлагаемый TOC)
1. **Problem** (факты, источники, карта)
2. **Solution: What is VODeco** (определение, SDG‑alignment)
3. **Ecosystem Layers** (12 слоёв, с подсказками)
4. **Data & Trust** (источники данных, верификация, аудит)
5. **Digital Twins** (объекты, паспорта, жизненный цикл)
6. **Economy** (TokenHub, инвестиции, KPI‑контракты)
7. **Token** (эволюция фазами, “ключ, а не вода”)
8. **Staking** (пулы, IRR, примеры расчётов)
9. **DAO Governance** (процессы, роли, комплаенс)
10. **Cabinets / Roles** (7 кабинетов и сценарии)
11. **Roadmap** (0–6 / 6–18 / 18–36 мес)
12. **Appendix** (глоссарий, ссылки на код и документы)

## 3) Интерактивные блоки (минимальный набор)
### A) Карта (Globe/Map)
- Слои: водные объекты / инфраструктура / проекты / инвестиции / датчики.
- Фильтры по типу/категории/стране/качеству.
- Источник MVP: `/api/water-resources?external=true`.

### B) Карточки проектов (TokenHub)
- Параметры: IRR, target/current, milestones, risk, region.
- Источник MVP: `/api/projects` + данные из `prisma/seed.ts`.

### C) Калькуляторы
- Покупка токенов (discount, savings).
- Стейкинг (APY по сроку/типу, rewards).
- Возврат проекта (IRR, duration).

Источник MVP: `vod-eco-mvp/src/lib/tokenomics/calculations.ts`.

### D) “Phases” (эволюция токена)
Переключатель фаз 1–4, где каждая фаза:
- что включает функционально;
- какие риски/ограничения;
- какие пререквизиты (данные, смарт‑контракты, аудит).

## 4) Принцип цитирования (доказуемость)
Каждый тезис должен иметь:
- ссылку на документ в репозитории **или**
- ссылку на участок кода (эндпоинт/модель).

Для интерактивной версии — “popover источника”.

## 5) Важный момент: 2 режима токеномики
Чтобы не было противоречий:
- **MVP mode** (упрощённый supply/цена, как в коде) — для демонстрации интерфейсов.
- **Tokenomics v2 mode** (water‑backed, фазы эмиссии, стабилизация) — для WhitePaper.

На странице: переключатель режима, который меняет подписи/формулы/константы.
