# VODeco Canon Docs (единый “канон” по экосистеме)

Этот набор документов собирает в одном месте **максимально полный и детализированный** контекст по VODeco: экосистема, механики, функции, токеномика, роли, флоу, архитектура и сопоставление с текущей реализацией (MVP).

## Что считается источником правды (Source of Truth)

### Реализовано в коде (MVP)
- `vod-eco-mvp/` — Next.js MVP (страницы, API, Prisma-модели, сиды, базовая токеномика/стейкинг).
- **Модель данных**: `vod-eco-mvp/prisma/schema.prisma`
- **Сиды/демо-экономика и проекты**: `vod-eco-mvp/prisma/seed.ts`
- **API**: `vod-eco-mvp/app/api/**`
- **Токеномика (вычисления в коде)**: `vod-eco-mvp/src/lib/tokenomics/calculations.ts`
- **Интеграции данных (WorldBank/USGS/OSM/News)**: `vod-eco-mvp/src/lib/api/**` + `vod-eco-mvp/README_API*.md`

### Концепт/продуктовые документы
- `Описание_экосистемы_VODeco.md` — слои (12 уровней), продукты, кабинеты, TokenHub-карточки.
- `архитектура и дополнения.txt` — расширенная онтология/архитектурная вертикаль, флоу “реальность → данные → доверие → экономика → DAO → обратная связь”.
- `Платформа VOD_документы.txt` — развернутая модель взаимодействия субъектов, роли токена, сценарии гос/инвест/общество.
- `vodeco_new/docs/TOKENOMICS_V2*.md` — токеномика v2 (варианты вода/водород), фазы, эмиссия, стабилизация.

## Навигация
- `01_Экосистема_и_миссия.md` — миссия, цели, рамки (SDG/ESG), позиционирование без “продажи воды”.
- `02_Архитектура_слоями.md` — уровни/слои, доменная архитектура, ключевой контур обратной связи.
- `03_Сущности_и_роли_кабинеты.md` — акторы, кабинеты/интерфейсы, права/обязанности, типовые флоу.
- `04_Продукты_и_модули.md` — продукты (VOD Check, Digital Twins, AI, DAO, TokenHub, SDK), что делает каждый модуль.
- `05_Токеномика_и_экономика.md` — модели VOD (фазы, выпуск, цена/якорь), сопоставление с MVP-калькуляциями.
- `06_Инвестиции_стейкинг_TokenHub.md` — стейкинг/IRR/пулы, механика “проектов”, как это реализовано в MVP.
- `07_DAO_управление_и_комплаенс.md` — governance, предложения, делегирование, аудит, институциональная версия.
- `08_Данные_реестры_верификация_ореклы.md` — источники данных, верификация, внешние API, модель доверия.
- `09_Текущая_реализация_MVP_карта.md` — карта текущих эндпоинтов/страниц/моделей БД, что уже работает.
- `10_Глоссарий.md` — определения (VOD, TokenHub, цифровой двойник, data anchoring, IRR, APY, кабинеты, субъекты).
- `11_Спецификация_интерактивного_WhitePaper.md` — структура интерактивной страницы (scroll-story, блоки, калькуляторы, интерактивы).
