# UX/UI архитектура и стиль (канон)

Цель: сделать UX максимально понятным для разных ролей и сценариев.

---

## 1) Принципы UX
- **Роль‑ориентированный UX**: пользователь сразу выбирает свою роль.
- **Прозрачность**: всегда видны прогресс, статус, метрики.
- **Единая карта мира** как главный навигационный центр.
- **Контекстные действия**: меньше сложных меню, больше быстрых действий.

---

## 2) Главные контуры UI
1. **Onboarding** (выбор роли → персональный дашборд)
2. **Globe / Map** (объекты, проекты, источники, фильтры)
3. **Social Feed** (обсуждения, ветки, миссии)
4. **TokenHub** (проекты, IRR, стейкинг)
5. **Data Lab** (данные, исследования, публикации)
6. **DAO** (предложения, голосование, аудит)

---

## 3) Минимальные UX‑паттерны
- глобальная панель фильтров для карты;
- popup карточка объекта → расширенная карточка ниже карты;
- единый язык статусов (draft / active / verified / archived);
- прозрачная система наград и прогресса.

---

## 4) Стиль и визуальные изменения
- акцент на **чистых градиентах** (вода/экология);
- снижение «неоновой нагрузки» для гос/институциональных ролей;
- модульная типографика (доступность, читаемость);
- минимализм для дашбордов, эмоциональность — для onboarding.

---

## 5) Роль‑специфические UI
- **Активист**: быстрые миссии + награды.
- **Исследователь**: data‑дашборды и публикации.
- **Инвестор**: IRR/риск, отчётность.
- **Гос/НКО**: комплаенс и KPI.
