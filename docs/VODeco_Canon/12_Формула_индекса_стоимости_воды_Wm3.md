# Формула индекса стоимости воды `W_m3` (канон)

Цель: получить **единую глобальную ценовую якорную единицу** для 1 м³ воды.  
`W_m3` — это **индекс**, а не фиатная цена. Он используется как base‑anchor для WTR и приводит стоимость воды к сопоставимому стандарту по всему миру.

---

## 1) Базовое определение
`W_m3` рассчитывается как нормализованный индекс из нескольких блоков:

```
W_m3 = BaseEnergy + BaseTreatment + BaseScarcity + BaseQuality + BaseCarbon + BaseExternalities - BaseEfficiency
```

Где каждый блок — взвешенный и нормализованный показатель, приводимый к единой шкале (например 0..1 или 0..100).

---

## 2) Блоки индекса (детализация)

### A) Энергозатраты на добычу/доставку
```
BaseEnergy = k1 * (E_extraction + E_transport + E_pumping) / V
```
- `E_extraction` — энергия добычи/забора
- `E_transport` — энергия транспортировки
- `E_pumping` — энергия насосов/давления
- `V` — объём (м³)
- `k1` — нормировочный коэффициент

### B) Стоимость фильтрации/очистки
```
BaseTreatment = k2 * (C_pre + C_filtration + C_disinfection + C_lab) / V
```
Учитывает:
- предварительную очистку
- фильтрацию (мембраны, активированный уголь, ультрафильтрация)
- обеззараживание
- лабораторный контроль качества

### C) Дефицит и устойчивость источника
```
BaseScarcity = k3 * ScarcityIndex * RiskFactor
```
- ScarcityIndex — индекс дефицита/доступности воды региона
- RiskFactor — климатические риски (засухи, изменения осадков)

### D) Качество воды и сложность очистки
```
BaseQuality = k4 * ContaminationScore
```
`ContaminationScore` формируется из загрязнений:
- биологические
- химические
- промышленные
- тяжёлые металлы
- микропластик

### E) Углеродный след
```
BaseCarbon = k5 * CO2eq_per_m3
```
`CO2eq_per_m3` — эквивалент выбросов на 1 м³ (добыча + очистка + доставка).

### F) Экстерналии и экосистемные потери
```
BaseExternalities = k6 * EcosystemImpact
```
Учитывает:
- деградацию экосистем
- потери биоразнообразия
- снижение воспроизводимости источников

### G) Эффективность инфраструктуры (вычитается)
```
BaseEfficiency = k7 * EfficiencyIndex
```
Снижает индекс там, где:
- низкие потери в сетях
- высокий water‑recycle
- энергоэффективные технологии

---

## 3) Нормализация и глобальная единица
Чтобы привести индекс к единой величине:

```
W_m3 = Normalize(Σ блоков) * GlobalAnchor
```

`GlobalAnchor` устанавливает глобальную «среднюю» базу (по консенсусу DAO и международных источников).

---

## 4) Контур обновления
- Период пересчёта: 1 раз в квартал (можно 1 раз в 6 мес).
- Источники: публичные тарифы, отчёты ВОЗ/ООН/национальные базы.
- Прозрачный реестр данных: источники → расчёт → хэш → верификация.

---

## 5) Зачем это инвесторам
`W_m3` создаёт:
- **глобально единый базовый стандарт стоимости воды**;
- **сравнимость проектов в разных странах**;
- **прозрачный контур доверия**, независимый от локальной валюты.

---

## 6) Мини‑схема расчёта

```
ДАННЫЕ (энергия, фильтрация, загрязнение, дефицит, CO2)
            ↓
Нормализация по шкале (0..1)
            ↓
Взвешивание (k1..k7)
            ↓
Сумма блоков - эффективность
            ↓
Глобальный индекс W_m3
```

---

## 7) Ограничения и допущения
- `W_m3` — индекс, не обещание рыночной цены.
- Локальные тарифы могут отличаться, но **цель — выравнивание**.
- Верификация данных критична: без неё индекс невалиден.
