# Воронки продаж, аирдропов и партнёрств (канон)

Цель: описать единые воронки вовлечения для разных типов участников.

---

## 1) Воронка раннего пользователя (активист)
1. Онбординг → выбор роли  
2. Миссии/задания → прогресс  
3. Верификация участия → награды  
4. Реферальная программа → рост сообщества  

## 2) Воронка исследователя
1. Регистрация → профиль исследователя  
2. Публикация работы  
3. Верификация/peer‑review  
4. Награда VOD credits  
5. Доступ к расширенным датасетам  

## 3) Воронка инвестора
1. Доступ к TokenHub  
2. Аналитика проектов + IRR  
3. Проверка комплаенс/рисков  
4. Стейкинг или финансирование  
5. Отчёты и KPI  

## 4) Воронка партнёра/инфлюенсера
1. Присоединение к партнёрской программе  
2. Уникальные ссылки и промо  
3. Верификация активности  
4. Вознаграждение  

---

## 5) Требования для функции продаж (минимум)
- платежная интеграция (fiat/crypto);
- юридическая модель (utility vs governance);
- условия lock/vesting;
- система AML/KYC (для институциональных);
- публичная документация (tokenomics + risks);
- прозрачный отчёт о распределениях.
