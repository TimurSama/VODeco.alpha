# План доведения до релиза v1 (привлечение пользователей и инвесторов)

Цель: выпустить первую публичную версию, пригодную для масштабирования, онбординга пользователей и переговоров с инвесторами.

---

## 1) Продукт и UX (что должен уметь пользователь)
- Онбординг и выбор роли (единый сценарий входа).
- Профиль + публикации + достижения (прозрачная ценность участия).
- Лента + комментарии + модерация (социальная активность и контроль).
- Глобус/карта + карточки объектов воды (основной «вау‑эффект» и контекст).
- Страница токеномики + кошелёк VOD credits (экономический контур MVP).
- Миссии, ачивки, уровни, рефералы (геймификация и рост).

**Критерий готовности:** новый пользователь понимает ценность за 3–5 минут и может совершить ключевые действия: читать/писать/комментировать/участвовать в миссиях/видеть прогресс.

---

## 2) Контент и смыслы (для пользователей и инвесторов)
- Канонические документы: миссия, токеномика, IoT‑концепция, W_m3, роли.
- Витрина кейсов/проектов: понятные карточки с целями, этапами и метриками.
- Публичный Whitepaper (интерактив + PDF).
- Прозрачные FAQ: роль VOD credits и WTR, эмиссия, риски, этапы.

**Критерий готовности:** инвестор может понять модель за 10–15 минут без общения с командой.

---

## 3) Техническая готовность (прод‑качество)
- PostgreSQL + миграции Prisma.
- Мониторинг ошибок (Sentry/логирование).
- Безопасность API (JWT, ограничение ролей, базовые rate‑limits).
- Скорость страниц (оптимизация рендеринга, lazy‑load).
- Отказоустойчивость внешних API (news/rates/rss).

**Критерий готовности:** стабильная работа на проде и повторяемый деплой.

---

## 4) Рост и воронки (MVP‑привлечение)
- Airdrop/рефералы/социальные миссии (в рамках VOD credits).
- Партнёрские сценарии (НКО, лаборатории, экологические проекты).
- Формы заявок от исследователей/инженеров/организаций.
- Подготовленные лендинги для 3–5 сегментов ролей.

**Критерий готовности:** есть повторяемый сценарий входа пользователей и механика удержания.

---

## 5) Юридические и финансовые блоки (минимум для релиза)
- Дисклеймеры: MVP, не инвестиционный совет, риски.
- Политика конфиденциальности/Terms (минимальный юр‑пакет).
- Концепция токена WTR как будущего этапа (без обещаний доходности).
- Контакт для инвесторов и партнёров.

**Критерий готовности:** базовая юридическая чистота и отсутствие «инвест‑обещаний».

---

## 6) Публичный релиз (go‑to‑market)
- Пресс‑kit: миссия, цифры, скриншоты, FAQ.
- План запусков: первый анонс → первые миссии → первые партнёры.
- Метрики запуска: регистрация, retention D1/D7, конверсия в миссии.

**Критерий готовности:** релиз можно повторять и масштабировать.

---

- [ ] Полные публикации: типы/теги/вложения + фильтры.

- [ ] Документация и Whitepaper актуальны.
- [ ] Маркетинговые страницы и формы партнёров.
- [ ] Минимальный юридический пакет и дисклеймеры.
- [ ] Мониторинг/логирование + базовые метрики.
